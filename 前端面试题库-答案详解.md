## 第 1 题：闭包是什么？使用场景？优缺点？（出现 28 次）

### 定义
闭包是指一个函数能够访问其外部函数作用域中的变量，即使外部函数已经执行完毕。简单说就是：**函数 + 它所引用的外部变量环境 = 闭包**。

```javascript
function outer() {
  let count = 0;
  return function() {
    return ++count;
  }
}
const fn = outer();
fn(); // 1
fn(); // 2  —— outer 已经执行完了，但 count 依然存活
```

### 使用场景
- **防抖/节流**：用闭包保存定时器变量
- **模拟私有变量**：外部无法直接访问
- **函数柯里化**：`sum(1)(2)(3)`
- **回调函数/事件处理**：在回调中保留外部变量引用
- **for 循环 + setTimeout 经典题**：用 IIFE 闭包保存每次循环的 i

### 优缺点
**优点**：变量长期驻留内存可做缓存；实现数据私有化，避免全局变量污染

**缺点**：
- **内存泄漏风险**：闭包引用的变量不会被垃圾回收，使用不当会导致内存占用过高
- **变量状态不易追踪**：调试困难
- 解决方法：用完后 `fn = null` 手动释放引用

### 回答结构
定义 → 举个例子 → 使用场景 → 优缺点

## 第 2 题：Vue 3 响应式原理（出现 27 次）

### 核心思想
Vue 的双向绑定基于 **数据劫持 + 发布订阅模式**：
- **数据 → 视图**：数据变化时，自动通知依赖该数据的组件重新渲染
- **视图 → 数据**：用户输入时（如 v-model），通过事件监听把新值写回数据

### Vue 3 实现（Proxy）

```javascript
const proxy = new Proxy(obj, {
  get(target, key, receiver) {
    track();  // 收集依赖
    return Reflect.get(target, key, receiver);
  },
  set(target, key, value, receiver) {
    Reflect.set(target, key, value, receiver);
    trigger();  // 派发更新
    return true;
  }
});
```

完整流程：`创建响应式数据 → get 触发 track（依赖收集）→ set 触发 trigger（派发更新）→ effect 重新执行 → 视图更新`

### Proxy 相比 Vue 2 的 defineProperty 的优势

| 优势 | 说明 |
|------|------|
| 监听属性新增/删除 | 不再需要 Vue.set / Vue.delete |
| 监听数组下标赋值 | `arr[0] = 'new'` 可以触发更新 |
| 惰性响应式，性能更好 | 按需劫持，不需要递归遍历所有属性 |
| 支持更多数据结构 | 可以代理 Map、Set、WeakMap 等 |

### 加分点
- Proxy 配合 Reflect 保证 this 指向正确
- v-model 其实是 `:value` + `@input` 的语法糖

### 回答结构
原理（数据劫持+发布订阅）→ Vue 3 实现 → Vue 2 缺点 → Vue 3 如何改进

## 第 3 题：跨域（出现 25 次）

### 什么是跨域
**协议 + 域名 + 端口号**，三者任一不同就是跨域。

```
https://www.example.com:443/api
  协议     域名          端口  路径
```

注意：和请求参数没有关系；子域名也算不同域名。

### 为什么要限制
同源策略是**浏览器的安全机制**，防止网页 A 的脚本读取网页 B 的敏感数据。注意：跨域限制是**浏览器做的**，服务器其实已经收到请求并返回了，是浏览器拦截了响应。

### 解决方案

| 方案 | 谁来做 | 原理 |
|------|-------|------|
| **CORS** | 后端 | 服务器设置 `Access-Control-Allow-Origin` 响应头 |
| **代理服务器** | 前端/运维 | 开发用 Vite proxy，生产用 Nginx 反向代理 |
| **JSONP** | 前后端配合 | 利用 `<script>` 标签不受同源策略限制，只支持 GET |
| **postMessage** | 前端 | 用于两个窗口/iframe 之间通信 |
| **WebSocket** | 前后端 | WebSocket 协议本身不受同源策略限制 |

### CORS 详解
- 服务器通过设置 HTTP 响应头，告诉浏览器"我允许这个跨域请求"
- **简单请求**：GET/POST/HEAD + 常见 Content-Type → 直接发
- **预检请求（Preflight）**：PUT/DELETE 或自定义头 → 先发 OPTIONS 请求，允许后才发真正请求
- 现代项目中 80%+ 的跨域请求都是预检请求（因为用 JSON + Authorization 头）

## 第 5 题：原型与原型链（出现 24 次）

### 核心区别

| | `prototype` | `__proto__` |
|--|--|--|
| **谁有** | 只有**函数**有 | **所有对象**都有 |
| **是什么** | 函数的原型对象，存放共享方法 | 指向创建该对象的构造函数的 prototype |
| **方向** | 向下提供 → "我给别人用" | 向上查找 ↑ "我去找谁" |

```javascript
const p = new Person('小明');
p.__proto__ === Person.prototype  // true ✅
```

### 原型链

当访问一个对象的属性时，如果自身没有，就沿着 `__proto__` 一层层往上找，直到 `null`：

```
p → Person.prototype → Object.prototype → null
```

### 记忆口诀
- `prototype` 是"我给别人用的"，`__proto__` 是"我去找别人的"
- 箭头指向：`__proto__` 指向 `prototype`

### 就背这 3 条
1. `实例.__proto__ === 构造函数.prototype`
2. `子类.prototype.__proto__ === 父类.prototype`
3. `Object.prototype.__proto__ === null`

## 第 6 题：var/let/const 区别（出现 23 次）

| 维度 | var | let | const |
|------|-----|-----|-------|
| 作用域 | **函数**作用域 | **块级**作用域 | **块级**作用域 |
| 重复声明 | ✅ 可以 | ❌ 不行 | ❌ 不行 |
| 重新赋值 | ✅ 可以 | ✅ 可以 | ❌ 不行 |
| 变量提升 | ✅ 提升且初始化为 undefined | 提升但**不初始化**（暂时性死区） | 同 let |
| 挂载到 window | ✅ | ❌ | ❌ |

### 关键点
- var 是**函数作用域**（不是没有作用域）
- const 不能重新赋值，但**引用类型的内容可以修改**：`const obj = {}; obj.name = '张三' // ✅`
- **暂时性死区（TDZ）**：let/const 在声明前访问会报 ReferenceError

## 第 7 题：this 指向（出现 22 次）

### 核心规则

| 调用方式 | this 指向 |
|---------|----------|
| `obj.fn()` | `obj` |
| `fn()` 直接调用 | `window`（严格模式下 `undefined`） |
| 箭头函数 | **定义时**外层函数作用域的 this，永远不变 |
| `new Fn()` | 新创建的实例对象 |
| `call/apply/bind` | 手动指定的对象 |

### 终极口诀
- **普通函数**：看**点前面**是谁（没有点就是 window）
- **箭头函数**：看**定义时外面最近的函数**的 this（对象 `{}` 不算作用域，要找到函数或全局）

### 易错点
对象字面量的 `{}` **不是作用域**！箭头函数在对象里定义，this 是全局，不是那个对象。

## 第 8 题：事件循环 Event Loop（出现 22 次）

### 执行顺序
```
同步代码 → 微任务 → 宏任务
```

| 类型 | 代表 | 优先级 |
|------|------|--------|
| **同步代码** | 普通语句、console.log | 🥇 最先 |
| **微任务** | Promise.then、async/await 后面的代码 | 🥈 第二 |
| **宏任务** | setTimeout、setInterval | 🥉 最后 |

### 关键规则
- **每执行完一个宏任务，都要先清空微任务队列，再执行下一个宏任务**
- Promise 构造函数里的代码是**同步**的
- `await` 像一把刀，同一行是同步，**下面的代码被放入微任务队列**
- `<script setup>` 本身也是同步执行的

### 经典题答案

```javascript
console.log(1);                    // 同步 → 1
setTimeout(() => console.log(2), 0); // 宏任务
Promise.resolve().then(() => console.log(3)); // 微任务
console.log(4);                    // 同步 → 4
// 输出：1, 4, 3, 2
```

## 第 13 题：防抖和节流（出现 18 次）

### 防抖（Debounce）
多次触发**只执行最后一次**，等触发停止后才执行。

```javascript
function debounce(fn, delay) {
  let timer = null;
  return function(...args) {
    clearTimeout(timer);
    timer = setTimeout(() => fn.apply(this, args), delay);
  };
}
```

场景：搜索框输入、按钮防重复提交、窗口 resize

### 节流（Throttle）
固定时间间隔内**只执行一次**。

```javascript
function throttle(fn, interval) {
  let lastTime = 0;
  return function(...args) {
    const now = Date.now();
    if (now - lastTime >= interval) {
      lastTime = now;
      fn.apply(this, args);
    }
  };
}
```

场景：页面滚动、鼠标移动、射击游戏开枪

### 核心区别
- 防抖：等你**停下来**才执行（电梯关门）
- 节流：按**固定频率**执行（红绿灯）

## 第 14 题：浏览器缓存（出现 18 次）

### 流程
```
检查强缓存 → 没过期直接用（不发请求）
    ↓ 过期了
发请求走协商缓存 → 没变返回 304（用缓存）
    ↓ 变了
返回 200 + 新资源
```

### 强缓存（不发请求）
- `Cache-Control: max-age=3600`（优先级高）
- `Expires`（老方式）

### 协商缓存（发请求问服务器）
- `ETag` / `If-None-Match`（按内容哈希，**优先级高**）
- `Last-Modified` / `If-Modified-Since`（按修改时间，秒级精度）

## 第 15 题：深拷贝与浅拷贝（出现 17 次）

### 三者区别

| | 赋值 | 浅拷贝 | 深拷贝 |
|--|------|-------|--------|
| 第一层 | 共享 | **独立** | **独立** |
| 嵌套对象 | 共享 | **共享** | **独立** |

### 浅拷贝方法
`{ ...obj }` / `Object.assign({}, obj)` / `arr.slice()`

### 深拷贝方法
- **`structuredClone(obj)`**（2026 年首选，原生 API）
- `JSON.parse(JSON.stringify(obj))`（有缺陷：不支持函数/Date/undefined）
- 手写递归

```javascript
function deepClone(obj) {
  if (obj === null || typeof obj !== 'object') return obj;
  const result = Array.isArray(obj) ? [] : {};
  for (let key in obj) {
    if (obj.hasOwnProperty(key)) {
      result[key] = deepClone(obj[key]);
    }
  }
  return result;
}
```

## 第 16 题：Vue 3 生命周期（出现 17 次）

### Options API vs Composition API

| 阶段 | Options API | Composition API |
|------|------------|-----------------|
| 创建前 | beforeCreate | ❌ 不需要（setup 本身就是） |
| 创建后 | created | ❌ 不需要（setup 本身就是） |
| 挂载前 | beforeMount | `onBeforeMount` |
| 挂载后 | mounted | `onMounted` |
| 更新前 | beforeUpdate | `onBeforeUpdate` |
| 更新后 | updated | `onUpdated` |
| 卸载前 | beforeUnmount | `onBeforeUnmount` |
| 卸载后 | unmounted | `onUnmounted` |

### 关键点
- `<script setup>` 本身就等于 `beforeCreate` + `created`
- Vue 2 的 `beforeDestroy/destroyed` 在 Vue 3 改名为 `beforeUnmount/unmounted`
- Composition API 优势：逻辑按功能聚合、支持 Tree-shaking、用 composables 替代 mixins

## 第 18 题：TCP 三次握手 / 四次挥手（出现 16 次）

### 三次握手（建立连接）
```
客户端 ──SYN──→     服务器     "我要连接你"
客户端 ←──SYN+ACK── 服务器     "好的，我也要连你"
客户端 ──ACK──→     服务器     "收到，开始吧"
```

### 四次挥手（断开连接）
```
客户端 ──FIN──→     服务器     "我说完了"
客户端 ←──ACK──     服务器     "收到，但我还没说完"
客户端 ←──FIN──     服务器     "我也说完了"
客户端 ──ACK──→     服务器     "好的，再见"
```

### 为什么握手 3 次挥手 4 次
握手时 SYN+ACK 可以合并成一条；挥手时 ACK 和 FIN 不能合并（中间服务器可能还要发数据）。

### 标志位
SYN = 建连接，ACK = 确认收到，FIN = 断连接

### 记忆口诀
握手三次确认"都能通"，挥手四次因为"不同步"。

## 第 19 题：数组去重（出现 16 次）

```javascript
// 方法一：Set（最简洁）
[...new Set(arr)]

// 方法二：filter + indexOf
arr.filter((item, index) => arr.indexOf(item) === index)

// 方法三：reduce
arr.reduce((acc, item) => acc.includes(item) ? acc : [...acc, item], [])

// 方法四：Map（性能最好 O(n)）
const map = new Map();
arr.filter(item => !map.has(item) && map.set(item, true))
```

对象数组去重需要按指定字段判断。

## 第 20 题：HTTP vs HTTPS（出现 16 次）

| | HTTP | HTTPS |
|--|------|-------|
| 安全性 | 明文传输 | 加密传输 |
| 端口 | 80 | 443 |
| 证书 | 不需要 | 需要 SSL/TLS 证书 |

### HTTPS 加密过程
1. **非对称加密**交换随机密钥（公钥加密，私钥解密）
2. **对称加密**传输数据（用交换好的随机密钥）
3. **证书**由 CA 签发，验证服务器身份，防止中间人攻击

> 非对称加密保证密钥交换安全，对称加密保证数据传输效率。

## 第 24 题：Promise（出现 15 次）

### 三种状态
`pending`（等待）→ `fulfilled`（成功）或 `rejected`（失败），状态只能变一次。

### API 对比

| 方法 | 等几个 | 失败处理 |
|------|--------|---------|

| `Promise.race` | 等**第一个完成** | 谁先完成用谁 |
| `Promise.allSettled` | 等**全部完成** | 不管成败都返回 |
| `Promise.any` | 等**第一个成功** | 全失败才失败 |

## 第 27 题：new 的过程（出现 14 次）

### 4 个步骤

1. 创建一个空对象
2. 把对象的 `__proto__` 指向构造函数的 `prototype`
3. 把 `this` 绑定到这个对象上并执行构造函数
4. 如果构造函数返回了对象就用它，否则返回新创建的对象

### 手写 new

```javascript
function myNew(Constructor, ...args) {
  const obj = Object.create(Constructor.prototype); // ①②
  const result = Constructor.apply(obj, args);      // ③
  return result instanceof Object ? result : obj;   // ④
}
```

### 易错点
- 如果构造函数 return 了一个对象，new 的结果就是那个对象
- 如果 return 的是基本类型（数字/字符串），忽略它

## 第 28 题：事件捕获、冒泡、事件委托（出现 14 次）

### 事件传递三个阶段
1. **捕获**：从外到里（document → 父 → 子）
2. **目标**：到达目标元素
3. **冒泡**：从里到外（子 → 父 → document）

```
捕获（外→里）           冒泡（里→外）
document → body → div → button → div → body → document
```

### 事件委托
利用冒泡机制，把事件绑定在父元素上，通过 `e.target` 判断实际触发的子元素。

```javascript
document.getElementById('list').addEventListener('click', (e) => {
  if (e.target.tagName === 'LI') {
    console.log(e.target.textContent);
  }
});
```

### 优点
- 减少内存（只绑定一个事件）
- 动态新增的子元素也能生效
- 代码简洁

### 阻止
- `e.stopPropagation()` —— 阻止冒泡
- `e.preventDefault()` —— 阻止默认行为

## 第 29 题：call / apply / bind（出现 14 次）

### 区别

| | call | apply | bind |
|--|------|-------|------|
| 传参 | 逗号隔开 | **数组** | 逗号隔开 |
| 执行 | ✅ 立即 | ✅ 立即 | ❌ **返回新函数** |

### 记忆口诀
call 逗号隔开，apply 数组包起来，bind 不执行等你来。

### 手写 call

```javascript
Function.prototype.myCall = function(obj, ...args) {
  obj = obj || window;
  const key = Symbol();
  obj[key] = this;              // 把函数挂到目标对象上
  const result = obj[key](...args); // 用对象调用，this 就指向对象
  delete obj[key];              // 删掉临时属性
  return result;
};
```

**原理**：谁调用函数 this 就指向谁，把函数临时变成目标对象的方法来调用。

## 第 30 题：TCP vs UDP（出现 13 次）

| | TCP | UDP |
|--|-----|-----|
| 连接 | 需要三次握手 | 不需要，直接发 |
| 可靠性 | ✅ 保证到达、顺序、不丢包 | ❌ 可能丢包、乱序 |
| 速度 | 较慢 | **快** |
| 类比 | 快递（签收制） | 发传单（扔了就走） |

### 使用场景
- **TCP**：网页（HTTP）、文件下载、邮件 —— 不能丢数据
- **UDP**：视频直播、语音通话、游戏、DNS —— 快比完整重要

### 记忆
> TCP 可靠但慢，UDP 快但不靠谱。看视频卡一下没事（UDP），下载文件少一个字节不行（TCP）。

## 第 31 题：进程和线程（出现 13 次）

| | 进程 | 线程 |
|--|------|------|
| 定义 | 系统**分配资源**的最小单位 | 系统**执行任务**的最小单位 |
| 关系 | 一个进程包含**多个线程** | 线程属于某个进程 |
| 内存 | 进程之间**独立** | 同一进程的线程**共享内存** |
| 崩溃 | 一个进程崩了不影响其他 | 一个线程崩了**整个进程崩** |
| 类比 | 工厂 | 工厂里的工人 |

### 浏览器例子
- Chrome 每个标签页是一个**进程**（所以一个页面崩了不影响其他）
- 每个标签页内有多个**线程**：JS 线程、渲染线程、事件线程、定时器线程

## 第 33 题：重绘和回流（出现 13 次）

| | 回流（Reflow） | 重绘（Repaint） |
|--|---------------|----------------|
| 触发 | 元素**位置/大小**变了 | 元素**外观**变了（颜色/阴影） |
| 开销 | **大**（重新计算布局） | 较小（重新画像素） |
| 关系 | 回流一定触发重绘 | 重绘不一定触发回流 |

### 减少方法
- 用 `transform` 代替 `top/left`
- 批量修改样式用 class
- 避免频繁读取布局属性（offsetWidth 等）
- 用 `requestAnimationFrame`

### 记忆口诀
回流改布局（贵），重绘改外观（便宜）。动画用 transform，样式用 class。

## 第 34 题：XSS 和 CSRF（出现 13 次）

### XSS（跨站脚本攻击）
**注入恶意代码**到网页，窃取用户数据。

防范：转义用户输入、CSP、HttpOnly Cookie、不用 innerHTML

### CSRF（跨站请求伪造）
**冒用用户身份**（cookie），骗服务器执行操作。

防范：CSRF Token、SameSite Cookie、验证 Referer、关键操作二次确认

### 区别
- XSS = 注入代码（偷数据）
- CSRF = 冒用身份（骗操作）
