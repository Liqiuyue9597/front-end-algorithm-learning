🎨 flex 简写对比演示

3 个 flex 子项 A / B / C，B 的内容比 A 和 C 长很多，看在不同 flex 值下表现差异。

flex: 1   →   1 1 0% 完全平分容器空间，无视内容长度
A
BBBBBBBBBBBBBBBBBBBBBBBBBB
C
A、B、C 宽度完全相等。哪怕 B 的内容很长，也不会比别人宽——因为 basis: 0% 让所有 item 从 0 开始，按 grow=1 平分。
flex: auto   →   1 1 auto 先按内容撑开，剩余空间再按比例分
A
BBBBBBBBBBBBBBBBBBBBBBBBBB
C
B 明显比 A 和 C 宽很多——因为 basis: auto 让 item 先按内容尺寸定起点，B 的内容长就先占大头。
flex: none   →   0 0 auto 完全按内容尺寸，不放大也不缩小
A
BBBBBBBBBBBBBBBBBBBBBBBBBB
C
每个 item 紧贴内容，剩余空间空着不分。
flex: initial（默认值）   →   0 1 auto 按内容尺寸，能缩不能放
A
BBBBBBBBBBBBBBBBBBBBBBBBBB
C
和 none 看起来差不多，但容器变窄时 item 会被压缩（none 不会）。
flex: 0   →   0 1 0% basis 是 0，又不放大 → 缩到最小
A
BBBBBBBBBBBBBBBBBBBBBBBBBB
C
item 缩到内容的最小宽度（被 min-content 兜底），剩余空间全空出来。

📐 经典应用：左右定宽 + 中间自适应（三栏布局）

把窗口拉宽拉窄看效果——左右两栏永远 150px，中间自动伸缩。

左 150px（flex-shrink: 0）
中间 flex: 1（自适应）
右 150px（flex-shrink: 0）
核心 CSS： 
.left, .right { width: 150px; flex-shrink: 0; }
.center { flex: 1; }

🎛️ 交互式调节 grow / shrink / basis

拖动滑条改变 B 的 flex 值，观察 B 相对 A 和 C 的变化。

flex-grow 1
flex-shrink 1
flex-basis 0px
当前 B 的 flex 值：1 1 0px
A (flex: 1)
B (动态)
C (flex: 1)
试试这些组合：
  • grow=2, shrink=1, basis=0 → B 拿到 A 和 C 两倍的剩余空间
  • grow=0, shrink=1, basis=200 → B 固定 200px（除非空间不够才被压缩）
  • grow=1, shrink=0, basis=300 → B 至少 300px，且容器变窄也不缩
